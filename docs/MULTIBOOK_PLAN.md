LeviLaser EasyAppointments Multibook Roadmap

## Plan A — мульти-услуги и стабильный UI
- [ ] A0 Инфраструктура
  - Настроить SSH-доступ с VPS к GitHub.
  - Включить git-игнор для чувствительных директорий (.env, storage, db dumps, vendor и т.п.).
- [ ] A1 Стабильный baseline
  - Откатить booking_type_step.php и booking.js к оригинальной версии из upstream-образа alextselegidis/easyappointments.
  - Проверить, что single-service бронирование работает так же, как в ванильном EA.
- [ ] A2 Мульти-услуги — первый шаг
  - Добавить чекбокс-UI для выбора нескольких услуг, без сложной фильтрации мастеров.
  - Подключить этот UI к уже существующему бэкенду (services[] + total_duration/total_price).
- [ ] A3 Фильтрация мастеров
  - Построить простые мапы service→providers и provider→services на фронтенде.
  - При выборе набора услуг показывать только тех мастеров, которые умеют все выбранные услуги.
- [ ] A4 Защита от невозможных комбинаций
  - Если новый набор услуг не может быть обслужен ни одним мастером, откатывать выбор и подсвечивать проблему.
  - В UI блёрить или дизейблить услуги, которые несовместимы с уже выбранными (ни один мастер не умеет весь набор).

## Git workflow / политика коммитов и пушей
- Основная ветка: main.
- Для крупных изменений использовать feature-ветки (например, feature/multibook-ui-step1).
- Каждый логический шаг — отдельный коммит с понятным сообщением.
- Не коммитить: .env, storage/, db/, vendor/, node_modules/, любые архивы/логи/дампы.
- После успешного теста на сервере — `git push origin <ветка>`.
- При работе с Codex: после каждого крупного шага из плана — коммит + пуш для удобного отката.
